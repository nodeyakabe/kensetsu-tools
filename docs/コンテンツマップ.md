# 建設業ツール工房 コンテンツマップ（記事キュー）

> 最終更新: 2026-07-07
> 目的: 3商品それぞれに記事クラスタを作り、内部リンクで束ねて「検索で上位＋AIに引用」の両取りを狙う。手間はClaude（内容）→Claude Code（実装）で自動化されている前提。キューとして上から順に指示書化して流す。
> 設計思想: 各記事は **① 結論先出し ② 数値・条件を構造化 ③ 法令などの根拠明示** の型で書く（AI Overview / ChatGPT / Perplexity に引用されやすくするAEO対策）。狙うのは実需クエリのみ。教習・受験など「客にならない層」のクエリは狙わない。

> **進捗管理**: 各記事表の「状態」列を、記事公開のたびに更新する（`未着手` → `公開済み（slug）`）。記事を実装した Claude Code が、その記事に対応する行の状態を1行だけ書き換えること。それ以外の記述は変更しない。

1. **安全台帳クラスタ**（通年需要・今すぐ動かせる・実証済みの効く型と同ジャンル）→ 最優先で厚くする

## 運用ルール（全記事共通）

1. **1本ずつ指示書化して流す**（着手確認§0つき）。既存の指示書テンプレート（本文完成形＋スキーマ踏襲＋リンク安全策）を踏襲。
2. **AEO型を守る**: 冒頭で結論を言い切る／数値・条件は表や箇条書きで構造化／法令の条番号など根拠を示す。
3. **カニバリ回避**: 新記事を出す前に、既存記事と狙うクエリが被っていないか確認。被る場合は統合か、役割を分けて相互リンク。
4. **内部リンク**: 各記事末から該当商品ページ（実在時のみ）＋同クラスタの関連記事1〜2本へ。束ね記事はクラスタ全記事へ。
5. **建設番頭リンクの注意**: 納品可能になるまでは強い送客をしない（買われても配布できないため）。それまでは「開発中・近日」の言及に留める。
6. **効果測定**: 月次でSearch Consoleを見て、クリックが付いた記事の型を次の記事に反映。表示だけ多くクリック0の記事は追わない（2024年問題ガイドの教訓）。

## 検証の見どころ

- **最初のテストケース = 安全台帳クラスタ**。通年需要なので、確定申告期を待たず「記事→商品」が売上に変わるかを検証できる。ここで手応えがあれば、同じ方式を他クラスタにも自信を持って展開。
- 検証枠②系（建設番頭クラスタ）で、2026年問題の需要を"公式と競合しない角度"で拾えるかも並行して見る。
